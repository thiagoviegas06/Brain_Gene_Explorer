# Brain Gene Expression Explorer

Interactive 3D brain visualization of gene expression data mapped onto the 4S156Parcels atlas.
Built with [yabplot](https://github.com/...) + PyVista + trame.

---

## Directory layout

```
brain-viz-python/
├── app.py              # Main trame web app — run this
├── build_atlas.py      # One-time atlas setup (run before app.py)
├── data_loader.py      # Data loading utilities (imported by app.py)
├── requirements.txt    # Python dependencies
└── atlas_cache/        # Generated by build_atlas.py
    ├── cortical/       # atlas.csv (vertex map) + atlas.txt (LUT)
    │                   # — requires wb_command, built by build_atlas.py
    └── subcortical/    # One .vtk mesh per parcel + atlas_LUT.txt
                        # — already built, no wb_command needed
```

Data lives in sibling `subject_bundle*/` directories (not inside this repo):

```
subject_bundle/                          # GTEX-1117F
├── atlas_info/                          # NIfTI volume + parcel metadata CSV
├── cache/{naive,dlam,plam}/GTEX-1117F.npz
├── gene_lists/*.txt                     # Curated gene presets
└── parcel_order.json                    # Canonical 150-parcel row order

subject_bundle_GTEX-13OW8/              # GTEX-13OW8
└── cache/{naive,dlam,plam}/GTEX-13OW8.npz
```

## Setup

Before running `app.py`, make sure these pieces are in place:

1. Python with the packages in `requirements.txt`
2. The subject data directories beside this repo:
   - `subject_bundle/`
   - `subject_bundle_GTEX-13OW8/`
3. The atlas cache
   - `atlas_cache/subcortical/` is already included and ready to use
   - `atlas_cache/cortical/` is optional, but needed if you want cortical parcels to render with real colours instead of a grey shell

### 1. Install Python dependencies

```bash
pip install -r requirements.txt
```

This installs the app's Python-side stack:

- `yabplot`
- `pyvista[trame]`
- `trame`, `trame-vtk`, `trame-vuetify`
- `wslink`

### 2. Confirm the data folders exist

`app.py` expects the subject data to live next to `brain-viz-python/`, not inside it.

From the parent project directory, you should have:

```text
gene_expression/
├── brain-viz-python/
├── subject_bundle/
└── subject_bundle_GTEX-13OW8/
```

If those sibling folders are missing, the app will not be able to load the atlas alignment,
gene lists, or subject `.npz` files at startup.

### 3. Atlas setup (one-time)

#### Minimum setup to run the app

No extra install is needed for the subcortical atlas. The repo already includes:

- `atlas_cache/subcortical/*.vtk`
- `atlas_cache/subcortical/atlas_LUT.txt`

With just the Python dependencies and subject data folders in place, you can run `app.py`
immediately. The app will start and the subcortical structures will render correctly.

#### Optional setup for cortical colouring (requires Connectome Workbench)

`build_atlas.py` projects the volumetric NIfTI atlas onto the fsLR32k cortical surface
via `wb_command`. Without this step, the cortex renders as a plain grey shell, but all
subcortical structures still render correctly with real data.

**Install Connectome Workbench on macOS:**

```bash
# 1. Open the installer (already in ~/Downloads)
open ~/Downloads/workbench-macub-v2.1.0.dmg

# 2. Drag the Workbench app to /Applications/

# 3. Add wb_command to PATH (~/.zshrc or ~/.bash_profile)
echo 'export PATH="/Applications/workbench/bin_macosx64:$PATH"' >> ~/.zshrc
source ~/.zshrc

# 4. Verify
wb_command -version
```

Then build the cortical atlas cache:

```bash
python build_atlas.py
```

This is a one-time step and may take a few minutes.

### 4. Run the app

```bash
python app.py
# Open http://localhost:1234
```

If the cortical atlas has not been built yet, that is still okay:

- subcortical regions will render normally
- cortical surfaces will appear grey

The app shows four side-by-side viewports sharing a single camera — rotating in any
panel rotates all four simultaneously. Use mouse drag to rotate, scroll wheel to zoom.

### Drawer controls

**Data**

| Control | Options | Notes |
|---|---|---|
| **Gene** | 88-gene dropdown (filtered from `ahba_100hvg.txt`) | Instant — re-slices the in-memory array, no I/O |
| **Subject** | GTEX-1117F, GTEX-13OW8, GTEX-11DZ1, GTEX-1B996, GTEX-1JMPZ | Reloads NPZ for the selected subject |
| **Model** | naive / dlam / plam | Affects V2 and V3; reloads the NPZ |

**V2 Panel**

| Control | Options | Notes |
|---|---|---|
| **V2 mode** | Reconstruction / Residual | Residual = LORO prediction − harmonized ground truth; switches V2 to a diverging colormap |

**View**

| Control | Options | Notes |
|---|---|---|
| **Brain clip** | Full / Half | Half clips at x = 0, keeping the left hemisphere; camera snaps to lateral view |
| **Color scale** | Local / Shared / WBF | Local: per-panel auto-scale. Shared: joint scale across models. WBF: lock all panels to the V3 range |

**Opacity**

| Control | Range | Notes |
|---|---|---|
| **Cortex** slider | 0 → 1 | Cortex surface opacity — lower to reveal subcortical structures |
| **Subcortex** slider | 0 → 1 | Deep nuclei opacity (thalamus, basal ganglia, etc.) |
| **Cerebellum** slider | 0 → 1 | Cerebellar parcel opacity |

### Colour scheme

- Sequential colormap: `viridis`, range auto-scaled per panel (or shared, see Color scale control)
- Diverging colormap: `RdBu_r`, used for V2 in Residual mode and V3 in Residual mode with dlam/plam
- Unmapped parcels (outside mask or NaN): light grey
- Background: white

## Data views

Four panels are always shown simultaneously:

| Panel | .npz array | Mask | What it shows |
|---|---|---|---|
| **V1 GTEx Values** | `loro_truth_subject_raw` | `loro_eval_mask` | Raw GTEx measurements at LORO held-out parcels (~9 parcels) |
| **V2 Reconstruction** | `loro_fused_subject_h` | `loro_eval_mask` | Out-of-sample LORO model predictions at held-out parcels. In Residual mode: shows `loro_fused_subject_h − loro_truth_subject_h` on a diverging colormap; click tooltip breaks down Truth / Predicted / Δ |
| **V3 Whole-brain Fit** | `fullfit_subject_h` | `imputed_mask` ∪ `gtex_mask` | Dense whole-brain fit across all 150 parcels. In Residual mode (dlam/plam only): shows WBF − AHBA Reference (naive fullfit); click tooltip shows AHBA / WBF / Δ |
| **V4 AHBA Reference** | `fullfit_subject_h` (naive) | `imputed_mask` ∪ `gtex_mask` | Naive model whole-brain fit used as a fixed AHBA reference baseline for the V3 residual map |

V1 and V2 remap RH-labelled parcels to their nearest LH neighbour so everything
renders on the left hemisphere (consistent with the upstream pipeline). V3 and V4
render bilateral — the full-fit predictions are mirrored symmetrically.

V1 and V4 are model-independent. V2 and V3 change with the Model dropdown.

---

## Hemisphere handling

The upstream pipeline forces all MNI x-coordinates negative (left hemisphere) before
model fitting. Some GTEx samples therefore end up matched to an RH-labelled AHBA parcel
(the mirrored RH copy was nearest), even though the model treats them as LH.

For V1 and V2, `data_loader.remap_rh_to_lh_nearest()` flips each RH parcel centroid
(x → −x) and maps the value to the closest LH parcel in the same structure. This is
pre-computed once at startup via `data_loader.build_rh_to_lh_map()`.

## Atlas notes

- **156 parcels total** in the NIfTI: 100 cortical + 56 subcortical.
- **150 parcels in the .npz**: 6 parcels have no AHBA coverage and are excluded.

  | Excluded parcel | Structure |
  |---|---|
  | `Cerebellar_Region9` | subcortex |
  | `LH-MN` | subcortex |
  | `RH-EXA` | subcortex |
  | `RH-STH` | subcortex |
  | `RH-VeP` | subcortex |
  | `RH_SomMot_2` | cortex |

- The `.npz` row order follows `parcel_order.json` (alphabetical AHBA labels), **not**
  the atlas CSV `index` column. `data_loader.load_alignment()` handles this alignment.
- `LH-MN` and `RH-HN` are too small to form a 3D mesh at 1 mm MNI resolution and are
  dropped from the subcortical VTK cache — neither carries GTEx data.

## Code overview

### `data_loader.py`

| Function | Returns | Use |
|---|---|---|
| `load_alignment()` | `(labels, atlas_aligned)` | 150-row aligned atlas DataFrame; call once at startup |
| `load_subject(model, subject_id)` | `NpzFile` | Load all arrays for one subject/model |
| `get_gene_idx(npz, gene)` | `int` | Column index for a gene name |
| `build_value_dict(npz, atlas, array_key, mask_key, gene)` | `(cortex_dict, subcortex_dict)` | Generic builder for any view |
| `v1_gtex_input(npz, atlas, gene)` | `(cortex_dict, subcortex_dict)` | V1: raw GTEx at LORO held-out parcels |
| `v2_reconstruction(npz, atlas, gene)` | `(cortex_dict, subcortex_dict)` | V2: LORO predictions at all GTEx parcels |
| `v4_fullfit(npz, atlas, gene)` | `(cortex_dict, subcortex_dict)` | V3 (UI): full-fit predictions, all 150 parcels |
| `build_rh_to_lh_map(atlas_aligned)` | `dict[str, str]` | Pre-compute RH→LH nearest-neighbour map; call once at startup |
| `remap_rh_to_lh_nearest(c_dict, s_dict, rh_to_lh)` | `(cortex_dict, subcortex_dict)` | Remap RH parcel values to their nearest LH parcel |
| `load_gene_list(preset)` | `list[str]` | Load a gene preset from `gene_lists/` |

All value dicts use `{parcel_label: float}` — parcels outside the mask or with NaN
are omitted, and PyVista renders those as `nan_color` automatically.

### `app.py`

- Loads atlas alignment, RH→LH map, and subject NPZ once at startup. Naive-model NPZ is also kept in memory as the fixed AHBA reference for the V3 residual map.
- `_make_viewport(col, title, subtitle)` builds one viewport: cortex + subcortical meshes with NaN scalars. All four viewports share a single VTK camera object. Returns mesh actors and title/subtitle text actors for later mutation.
- `update_scene(c1, s1, c2, s2, c3, s3, c4, s4, gene, cbar_mode, sc, v2_is_residual, v3_is_residual)` updates scalar arrays and colour limits in-place — no actor teardown, efficient re-render. Switches V2/V3 between sequential (`viridis`) and diverging (`RdBu_r`) LUTs based on residual flags.
- `_reload_scene(c1, s1, c2, s2, c3, s3, gene, v2_is_residual)` orchestrates a full scene refresh: computes the V3 residual (WBF − naive reference) when applicable, updates V3 title/subtitle text actors, populates `_residual_truth`/`_residual_pred`/`_v3_residual_ref` lookup dicts used by the click tooltip, and calls `update_scene` + `_update_cbars`.
- `_on_click` picks the clicked parcel label and value from the active actor, then builds a simple or three-row tooltip depending on which panel was clicked and whether residual mode is active.
- State callbacks: `on_gene`, `on_subject`, `on_model`, `on_v2_mode`, `on_half_brain`, `on_cortex_alpha`, `on_subcortex_alpha`, `on_cerebellum_alpha`, `on_cbar_mode`, plus rotation counters (`rot_left/right/up/down`).
